## 7단계 힌트 시스템

AI 튜터는 직접 답을 주지 않고, 단계적으로 사고를 유도하는 7단계 힌트 시스템을 사용합니다.

### 단계별 설명

| 단계 | 아이콘 | 내용 | 예시 질문 |
|------|--------|------|----------|
| 1단계 | 📋 | **문제 이해 확인** | "이 문제에서 입력과 출력은 무엇인가요?" |
| 2단계 | 📎 | **관련 개념 연결** | "이 문제는 어떤 자료구조와 관련이 있을까요?" |
| 3단계 | 💡 | **핵심 아이디어 유도** | "해시맵을 사용하면 어떤 이점이 있을까요?" |
| 4단계 | 📝 | **의사코드 작성 유도** | "알고리즘의 전체 흐름을 한국어로 설명해보세요" |
| 5단계 | 🏗️ | **구조 설계 가이드** | "함수의 인자와 반환값을 정의해보세요" |
| 6단계 | ✍️ | **코드 작성 지원** | "핵심 로직을 코드로 작성해보세요" |
| 7단계 | 📊 | **복잡도 분석 및 최적화** | "시간 복잡도를 분석하고 최적화 방안을 생각해보세요" |

### 사용 예시

**사용자**: "Two Sum 문제 힌트 알려줘"

**AI 튜터 응답**:
```
📋 **1단계: 문제 이해**
- 정수 배열 nums와 목표값 target이 주어집니다
- 합이 target이 되는 두 숫자의 인덱스를 찾아야 합니다

📎 **2단계: 관련 개념**
- 이 문제는 "탐색" 문제입니다
- 해시맵(딕셔너리)을 활용하면 효율적으로 풀 수 있습니다

💡 **3단계: 핵심 아이디어**
각 숫자를 순회하면서, 현재 숫자의 "보수"(target - 현재 숫자)가 
이미 본 숫자들 중에 있는지 확인하면 어떨까요?

직접 생각해보세요! 더 힌트가 필요하면 말씀해주세요.
```

## 코드 리뷰 기능

AI 튜터는 작성한 코드를 분석하여 복잡도 분석과 최적화 제안을 제공합니다.

### 분석 항목

1. **시간 복잡도 분석**
   - 현재 알고리즘의 Big-O 표기법 분석
   - 최적 복잡도와 비교

2. **공간 복잡도 분석**
   - 추가 메모리 사용량 분석
   - 인플레이스 최적화 가능 여부

3. **코드 품질**
   - 변수 명명 규칙
   - 코드 가독성
   - 중복 코드 탐지

4. **최적화 제안**
   - 더 효율적인 알고리즘 제안
   - 자료구조 개선 방안

### 사용 예시

**사용자**: 
```python
def two_sum(nums, target):
    for i in range(len(nums)):
        for j in range(i+1, len(nums)):
            if nums[i] + nums[j] == target:
                return [i, j]
    return []
```
"이 코드 리뷰해줘"

**AI 튜터 응답**:
```
📊 **코드 리뷰 결과**

**시간 복잡도**: O(n²)
- 중첩 for 루프로 인해 이차 시간 복잡도
- n이 커지면 성능 저하가 심각해집니다

**공간 복잡도**: O(1)
- 추가 메모리 사용 없음 (좋음!)

🔧 **최적화 제안**:
해시맵을 사용하면 O(n)으로 개선 가능합니다:

def two_sum(nums, target):
    seen = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
    return []

개선점:
- 시간 복잡도: O(n²) → O(n)
- 공간 복잡도: O(1) → O(n) (트레이드오프)
```

## 디버깅 도움

코드에서 발생하는 오류의 원인을 분석하고 해결 방법을 제시합니다.

### 지원 오류 유형

| 오류 유형 | 설명 | 일반적인 원인 |
|----------|------|--------------|
| IndexError | 배열 인덱스 초과 | 경계 조건 미처리, off-by-one 에러 |
| RecursionError | 재귀 깊이 초과 | 종료 조건 누락, 무한 재귀 |
| TimeoutError | 시간 초과 | 비효율적 알고리즘, 무한 루프 |
| WrongAnswer | 오답 | 엣지 케이스 미처리, 로직 오류 |
| TypeError | 타입 오류 | 잘못된 타입 연산 |
| KeyError | 키 오류 | 딕셔너리 키 미존재 |

### 사용 예시

**사용자**:
```python
def find_max(arr):
    max_val = arr[0]
    for i in range(len(arr)):
        if arr[i+1] > max_val:  # IndexError 발생!
            max_val = arr[i+1]
    return max_val
```
"IndexError가 발생해. 도와줘"

**AI 튜터 응답**:
```
🔍 **오류 분석**

**오류 유형**: IndexError - 배열 범위 초과

**원인**: 
- for 루프에서 i가 0부터 len(arr)-1까지 순회합니다
- arr[i+1]에서 i가 마지막 인덱스일 때 범위를 초과합니다
- 예: 배열 길이가 5면 i=4일 때 arr[5] 접근 시도

**해결 방법**:
def find_max(arr):
    max_val = arr[0]
    for i in range(len(arr) - 1):  # 수정: -1 추가
        if arr[i+1] > max_val:
            max_val = arr[i+1]
    return max_val

💡 **팁**: 또는 더 파이썬스러운 방식으로:
def find_max(arr):
    return max(arr)  # 내장 함수 활용
```

## 문제 추천

사용자의 실력과 약점을 분석하여 맞춤형 문제를 추천합니다.

### 추천 기준

1. **난이도 기반**
   - 현재 실력에 맞는 적정 난이도 문제
   - 성장을 위한 살짝 어려운 문제

2. **약점 기반**
   - 자주 틀리는 유형 분석
   - 연습이 필요한 알고리즘 패턴

3. **학습 진도 기반**
   - 아직 시도하지 않은 문제
   - 복습이 필요한 문제

### 사용 예시

**사용자**: "DP 문제 추천해줘"

**AI 튜터 응답**:
```
📚 **DP 문제 추천**

분석 결과, DP 카테고리에서 다음 문제들을 추천드립니다:

🟢 **Easy** (기초 다지기)
1. Climbing Stairs - 기본 DP 패턴 학습
2. House Robber - 1D DP 응용

🟡 **Medium** (실력 향상)
3. Longest Increasing Subsequence - LIS 패턴
4. Coin Change - 배낭 문제 변형

🔴 **Hard** (도전!)
5. Edit Distance - 2D DP
6. Longest Valid Parentheses - 스택 + DP 결합

💡 **추천**: 현재 실력으로 "Coin Change"부터 시작하면 좋겠습니다!
```

## 효과적인 사용법

### Do's (권장사항)

✅ **구체적으로 질문하기**
- 좋음: "Two Sum 문제에서 해시맵 사용법을 모르겠어"
- 나쁨: "문제 풀어줘"

✅ **단계별로 학습하기**
- 힌트를 받으면 먼저 직접 생각해보기
- 막히면 다음 단계 힌트 요청

✅ **코드를 공유하기**
- 작성한 코드와 함께 질문하면 더 정확한 도움

✅ **오류 메시지 포함하기**
- 발생한 오류 메시지와 함께 질문

### Don'ts (주의사항)

❌ **정답을 직접 요청하지 않기**
- AI 튜터는 학습을 위해 직접 정답을 제공하지 않습니다

❌ **너무 모호한 질문 피하기**
- "어떻게 해?" 보다 "이 부분이 이해가 안 돼" 가 좋음

## 지원 알고리즘 패턴

AI 튜터는 45개의 알고리즘 패턴에 대해 학습 지원을 제공합니다:

- **기본**: Two Pointers, Sliding Window, Binary Search
- **트리/그래프**: BFS, DFS, Topological Sort
- **DP**: Memoization, Tabulation, State Machine
- **고급**: Segment Tree, Union-Find, Trie
- **문자열**: KMP, Rabin-Karp, Z-Algorithm

## FAQ

**Q: AI 튜터가 정답을 안 알려줘요**
A: AI 튜터는 학습 효과를 위해 직접 정답을 제공하지 않습니다. 단계별 힌트를 통해 스스로 해결할 수 있도록 유도합니다.

**Q: 더 자세한 힌트가 필요해요**
A: "더 힌트 줘" 또는 "다음 단계 힌트 알려줘"라고 말씀해주세요.

**Q: 특정 알고리즘 패턴을 배우고 싶어요**
A: "[패턴명] 설명해줘" 또는 "패턴" 페이지에서 직접 학습할 수 있습니다.
