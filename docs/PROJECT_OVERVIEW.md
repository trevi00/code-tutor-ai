# Code Tutor AI - 프로젝트 개요

## AI 기반 Python 알고리즘 & 자료구조 학습 플랫폼

**버전**: 3.0
**최종 수정**: 2025-12-26

---

## 1. 프로젝트 정의

### 1.1 비전
> 한국어 AI 튜터와 함께하는 맞춤형 알고리즘 학습 플랫폼

### 1.2 미션
허깅페이스의 한국어 LLM을 활용하여 초보 개발자들이 알고리즘과 자료구조를 효과적으로 학습할 수 있는 AI 기반 맞춤형 교육 플랫폼 구축

### 1.3 핵심 가치
- **개인화**: AI가 학습 패턴을 분석하여 맞춤형 문제 추천
- **대화형**: 자연어로 질문하고 이해하기 쉬운 답변 제공
- **실용성**: 한국 코딩테스트(카카오, 삼성, 네이버) 대비
- **접근성**: 오픈소스 기반 비용 효율적 솔루션

## 3. 핵심 기능

### 3.1 기능 우선순위

| 우선순위 | 기능 | 설명 | Phase |
|----------|------|------|-------|
| **P0** | AI 튜터 대화 | LLM 기반 자연어 Q&A | MVP |
| **P0** | 코드 리뷰 | 복잡도 분석, 개선 제안 | MVP |
| **P1** | 문제 풀이 | 코드 제출, 자동 채점 | MVP |
| **P1** | 코드 실행 | Docker 샌드박스 | MVP |
| **P1** | 맞춤형 추천 | NCF 기반 문제 추천 | Phase 2 |
| **P2** | 학습 진도 | 대시보드, 통계 | Phase 2 |
| **P2** | 유사 코드 검색 | CodeBERT 임베딩 | Phase 3 |
| **P2** | 성과 예측 | LSTM 시계열 분석 | Phase 3 |

### 3.2 핵심 사용자 시나리오

```
1. 로그인 → 대시보드에서 오늘의 추천 문제 확인
2. 문제 선택 → 설명 읽고 코드 작성
3. 막히면 AI 튜터에게 질문 → 힌트 받기
4. 코드 제출 → 자동 채점 결과 확인
5. AI 코드 리뷰 → 개선점 학습
6. 다음 문제 추천 받기
```

## 4. 기술 스택 요약

### 4.1 아키텍처
- **설계 패턴**: DDD (Domain-Driven Design) + 헥사고날 아키텍처
- **상세 문서**: [DDD_ARCHITECTURE.md](./DDD_ARCHITECTURE.md)

### 4.2 기술 선택

| 영역 | 기술 | 선택 이유 |
|------|------|-----------|
| **Backend** | FastAPI + Python 3.11 | 비동기, 타입힌트, ML 생태계 |
| **Frontend** | React 18 + TypeScript | 컴포넌트 재사용, 타입 안전성 |
| **Database** | PostgreSQL + Redis | 안정성, 캐싱 |
| **LLM** | EEVE-Korean-2.8B | 한국어 특화, 오픈소스 |
| **ML** | PyTorch + HuggingFace | 유연성, 모델 다양성 |
| **Container** | Docker + Compose | 일관된 환경, 샌드박스 |

### 4.3 딥러닝 모델 스택

| 모델 | 용도 | 크기 | 메모리 |
|------|------|------|--------|
| EEVE-Korean-2.8B | AI 튜터 | 2.8B | 2.5GB (4-bit) |
| DistilCodeBERT | 코드 임베딩 | 66M | 1.2GB (8-bit) |
| NCF (Custom) | 문제 추천 | 100K | 5MB (CPU) |
| LSTM (Custom) | 성과 예측 | 500K | 2MB (CPU) |

### 4.4 하드웨어 제약

```
GPU: NVIDIA RTX 4050 (4GB VRAM)
전략: 동적 모델 로딩 + 양자화 + CPU 오프로딩
```

## 5. 프로젝트 범위

### 5.1 In Scope (범위 내)
- Python 알고리즘 문제 (25개 패턴, 100+ 문제)
- 한국어 AI 튜터
- 웹 기반 코드 에디터
- Docker 샌드박스 실행
- 개인 학습 통계

### 5.2 Out of Scope (범위 외)
- 다른 언어 (JavaScript, Java) - 향후 확장
- 모바일 앱 - 웹 반응형으로 대체
- 실시간 멀티플레이어
- 외부 API 연동 (LeetCode, BOJ)

## 6. 성공 지표 (KPI)

### 6.1 MVP 목표 (4주)

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| AI 응답 시간 | < 5초 | 서버 로그 |

| 일일 활성 사용자 | 10명 | Analytics |
| 문제 완료율 | > 30% | DB 집계 |

### 6.2 Phase 2 목표 (8주)

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|

| 사용자 만족도 | NPS > 40 | 설문 |
| 주간 리텐션 | > 60% | Cohort 분석 |

## 7. 개발 로드맵

### Phase 1: MVP (4주)
```
Week 1: 인프라 + 인증
  - Docker 환경 구축
  - PostgreSQL + Redis 설정
  - JWT 인증 구현

Week 2: 핵심 기능
  - 문제 CRUD
  - 코드 실행 샌드박스
  - 제출 및 채점

Week 3: AI 통합
  - EEVE-Korean 연동
  - AI 튜터 대화
  - 코드 리뷰

Week 4: 프론트엔드 + 배포
  - React UI 완성
  - E2E 테스트
  - 프로덕션 배포
```

### Phase 2: 고도화 (4주)
```
Week 5-6: 추천 시스템
  - NCF 모델 학습
  - 맞춤형 문제 추천

Week 7-8: 학습 분석
  - 대시보드
  - 진도 추적
  - 성과 예측
```

### Phase 3: 확장 (4주)
```
Week 9-10: 코드 분석
  - CodeBERT 임베딩
  - 유사 코드 검색
  - 품질 분석

Week 11-12: 최적화
  - 성능 튜닝
  - 사용자 피드백 반영
  - 문서화
```
